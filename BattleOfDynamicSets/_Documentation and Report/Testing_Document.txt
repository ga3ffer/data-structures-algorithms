/*
 *										Analysis
 */
 
 
The runtime results of these four dynamic sets are comparable to the asymptotic analyses discussed in the Cormen textbook, lecture notes and in class. 

With the doubly linked list implementation, insertion was the slowest operation.  This is not surprising because it is O(n), and it is evident in the runtimes as the input file size is increased.  Searching for a key,  finding the predecessor, and successor are very inefficient in comparison to the other dynamic implementations.

Looking at the minimum and maximum operations, the doubly linked list set is still faster than the tree data structures.  The DLL is implemented using a circular list with a sentinel node, so finding the minimum and maximum element should be O(1) time, since the list is already sorted.

The runtimes for insertion in a SkipList can vary, since its a randomized algorithm.  Searching a skip list takes an expected O(lg n) time, due to the fact that it needs to drop down levels, until it finds the key searching for. 

Binary search trees and red black trees are similar, since both of them are a essentially binary search trees, with the difference that red black trees has additional properties. Insertion for a red black tree is more efficient however, since the worst case is O(lg n). A regular binary tree has a worst case of O(n), ex: inserting an already sorted set. Red black trees seem to be the fastest of all the 4 sets when it comes to insertion. 

Red black trees are the fastest and most efficient structures on large data sets.  It is faster than normal Binary Search Trees because it restructures, and recolours the tree upon the insertion of a new element, to stay balanced.  All the other data structures rank from skip lists (if probability says worst case is O(lg n)), then binary search trees (though the ranking for these two are close), and doubly linked lists.
